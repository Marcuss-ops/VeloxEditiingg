PR status check: "Pre-existing Test Watchlist". Concurrency group
`watchlist-${ref}` is independent of `ci-${ref}` so this gate runs
in parallel with `make verify`, `routing-invariants.yml`, and
`workspace-tests.yml`. Pull-request runs cancel-in-progress; pushes
to main run to completion.

Cache key uses only DataServer/go.sum (the watchlist does not test
anything in worker-agent-go or shared). timeout-minutes: 10 (4 tests
each ~3 s when green, comfortably gives latitude for SQLite-migration
plumbing and contention-burst paths).

Standard fail-fast (no continue-on-error): the user explicitly wants
these tests surfaced as PR status checks rather than hidden in
aggregate runs. The 4 tests are EXPECTED to go RED when this gate
lands; fixing them becomes the team's measurable work-in-progress.

Run-step also echoes a one-line watchlist summary at the end of the
run (PASS / FAIL counts per test) so PR reviewers can skim the
status check log without scrolling through verbose Go test output.

Verified locally: WATCHLIST gate run confirms all 4 specified tests
